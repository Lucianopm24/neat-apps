Flujo OAuth2

Neat usa el flujo Authorization Code. Tu app redirige al usuario a la pantalla de autorización, el usuario aprueba, y recibes un code que intercambias por un token.

1. Redirigir al usuario

GET https://neat.blue/oauth.html ?client_id=TU_CLIENT_ID &redirect_uri=https://tu-app.com/callback &scope=profile email &state=RANDOM_STRING

2. Intercambiar el code por token

POST https://neat-apps-b.vercel.app/oauth/token { "code": "EL_CODE", "clientId": "TU_CLIENT_ID", "clientSecret": "TU_CLIENT_SECRET", "redirectUri": "https://tu-app.com/callback" }

3. Obtener info del usuario

GET https://neat-apps-b.vercel.app/oauth/userinfo Authorization: Bearer ACCESS_TOKEN

PKCE — apps públicas (sin secret)recomendado para SPA / móvil

Si tu app no puede guardar un client_secret de forma segura — por ejemplo una SPA donde cualquiera puede abrir el código fuente, o una app móvil donde alguien podría inspeccionarla — usa PKCE en vez de mandar el secret. Marcá tu app como "pública" al crearla en el panel y seguí estos pasos en tu código:

1. Generar un code_verifier y su code_challenge

Antes de redirigir al usuario, tu app genera un secreto aleatorio de un solo uso (code_verifier) y lo guarda en memoria. Calcula su hash (code_challenge) y manda solo el hash a Neat — el secreto original nunca viaja por la red en este paso.

function base64url(buffer) { return btoa(String.fromCharCode(...new Uint8Array(buffer))) .replace(/\+/g, '-').replace(/\//g, '_').replace(/=+$/, ''); } function randomVerifier(len = 64) { const bytes = crypto.getRandomValues(new Uint8Array(len)); return base64url(bytes); } async function challengeFromVerifier(verifier) { const hash = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(verifier)); return base64url(hash); } const codeVerifier = randomVerifier(); const codeChallenge = await challengeFromVerifier(codeVerifier); // Guarda codeVerifier en memoria (ej. sessionStorage) — lo necesitas en el paso 3 sessionStorage.setItem('pkce_verifier', codeVerifier);

2. Redirigir al usuario con el code_challenge

GET https://neat.blue/oauth.html ?client_id=TU_CLIENT_ID &redirect_uri=https://tu-app.com/callback &scope=profile email &state=RANDOM_STRING &code_challenge=EL_CODE_CHALLENGE &code_challenge_method=S256

3. Intercambiar el code por token (sin secret)

En vez de clientSecret, mandás el code_verifier original que guardaste en el paso 1.

POST https://neat-apps-b.vercel.app/oauth/token { "code": "EL_CODE", "clientId": "TU_CLIENT_ID", "redirectUri": "https://tu-app.com/callback", "codeVerifier": "EL_CODE_VERIFIER_GUARDADO_EN_PASO_1" }

Neat vuelve a hashear ese code_verifier y lo compara contra el code_challenge que recibió en el paso 2. Si coinciden, te entrega el token — sin que tu app haya necesitado nunca un secret fijo.

Ejemplo en JavaScript

// Después de recibir el code en tu callback: const res = await fetch('https://neat-apps-b.vercel.app/oauth/token', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ code, clientId: 'TU_CLIENT_ID', clientSecret: 'TU_CLIENT_SECRET', redirectUri: 'https://tu-app.com/callback' }) }); const { access_token } = await res.json(); // Obtener perfil: const user = await fetch('https://neat-apps-b.vercel.app/oauth/userinfo', { headers: { Authorization: `Bearer ${access_token}` } }).then(r => r.json()); console.log(user.username, user.email);
